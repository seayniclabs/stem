What is Stem?

Stem is an MCP (Model Context Protocol) server that connects Apple Music to AI coding tools like Claude Code, Cursor, and any MCP-compatible client. Search the catalog, control playback, and manage playlists — all through natural language.

What Stem is

  • A bridge between your AI tools and Apple Music
  • Native macOS — built with Swift and MusicKit
  • Zero-config auth — one system prompt, no API keys
  • Privacy-first — no telemetry, no data leaves your Mac
  • Free and open source

What Stem is not

  • A music player — playback goes through Music.app
  • A replacement for the Music app
  • Cross-platform — macOS only (MusicKit requirement)
  • An AI model — your AI tool provides the intelligence
  • A streaming service — requires your own Apple Music subscription

Requirements

  • macOS 14 (Sonoma) or later
  • Apple Music subscription (for playback; catalog search works without one)
  • An MCP-compatible AI tool (Claude Code, Cursor, etc.)

Quickstart

  1. Install 
    $ brew install seayniclabs/tap/stem

    Or download the installer from the releases page.

  2. Setup 
    $ stem setup

  3. Connect to your AI tool 
    # Claude Code
$ claude mcp add stem -- /usr/local/bin/stem serve

# Or add to ~/.claude.json manually:
{
  "mcpServers": {
    "stem": {
      "command": "/usr/local/bin/stem",
      "args": ["serve"]
    }
  }
}
  4. Use it

    Ask your AI tool anything about Apple Music:

    "Search Apple Music for Tycho"
"What's currently playing?"
"Skip to the next track"
"Pause the music"

How It Works

Stem uses Apple's MusicKit framework to communicate with Apple Music. When your AI tool sends a command (like "search for Radiohead"), it calls Stem over the MCP protocol. Stem translates that into a MusicKit API call, gets the results from Apple, and returns them to your AI tool.

Your AI tool  ──stdio──>  Stem  ──MusicKit──>  Apple Music
(Claude, etc.)           (Swift)                 (catalog, library)
                            │
                            └──ApplicationMusicPlayer──>  Music.app
                                                          (playback)

All communication stays on your Mac. Stem doesn't phone home, collect analytics, or send data anywhere except directly to Apple's servers (the same way Music.app does).

Troubleshooting

Apple Music access denied

Go to System Settings → Privacy & Security → Media & Apple Music and enable access for Stem. Then run stem setup again.

Playback doesn't work

Playback requires an active Apple Music subscription. If you have Apple Music, make sure you're signed in on this Mac. Catalog search works without a subscription.

MCP server not connecting

After adding Stem to your AI tool's MCP config, restart the tool. Claude Code and Cursor require a restart to pick up new MCP servers.
